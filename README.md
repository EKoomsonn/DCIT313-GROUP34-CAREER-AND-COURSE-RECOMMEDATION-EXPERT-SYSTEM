# DCIT313 – Career & Course Recommendation Expert System

### Group 34

## Project Overview

This project implements a **Rule-Based Expert System** designed to assist students in selecting suitable academic programs based on their **academic strengths, interests, and skill levels**.

Many students struggle to choose the right university program due to limited career guidance, peer influence, or lack of awareness about the requirements of different fields. This system simulates the reasoning process of an academic advisor by using **IF–THEN rules** stored in a knowledge base.

The system collects user inputs such as:

* Strong academic subjects (Mathematics, Biology, English, etc.)
* Personal interests (Technology, Healthcare, Communication, etc.)
* Skill levels

Using **forward chaining inference**, the system evaluates the inputs against predefined rules and produces a **recommended course or academic program** along with a brief explanation.

---

## Technical Stack

**Logic Engine:**

* SWI-Prolog (Knowledge representation and inference rules)

**User Interface:**

* Python with `pyswip` library (Connects Python to Prolog)

**Version Control:**

* GitHub

---

## System Architecture

The project follows a modular Knowledge-Based System architecture:

```
/knowledge_base
    rules.pl        -> Prolog facts and inference rules

/interface
    main.py         -> Python interface for user interaction

/docs
    documentation and knowledge acquisition sources
```

The **Prolog knowledge base stores the intelligence of the system**, while the **Python interface handles user interaction and queries the inference engine**.

---

## Project Purpose

The goal of this expert system is to:

* Provide **career guidance for students**
* Help students **choose suitable academic programs**
* Simulate the reasoning process of **academic advisors**
* Reduce cases of **poor course selection**

## Roles and Responsibilities

### Project Manager

Coordinates the project, manages timelines, oversees collaboration, ensures the repository structure is correct, and prepares the group for presentation and submission.

### Domain Expert

Provides expert knowledge about academic programs, required skills, and career paths. Ensures that the recommendations generated by the system are realistic and academically valid.

### Knowledge Engineer

Transforms the domain expert's knowledge into **formal IF–THEN rules** and implements them in the Prolog knowledge base.

### Programmers

Develop the system components, including:

* Python user interface
* Integration between Python and Prolog
* System functionality and debugging

### End User

Tests the system using sample inputs and provides feedback on usability, accuracy, and effectiveness of the recommendations.

---

## Example Rule

Example rule implemented in the knowledge base:

```
IF strong in Mathematics AND enjoys problem-solving
THEN recommend Engineering
```

## Expected Output

* Recommended academic program
* Explanation of why the program was recommended
